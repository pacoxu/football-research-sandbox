# Security Policy

更新时间：2026-07-12

本仓库主要维护公开足球研究资料和静态站点代码。安全与风险报告重点包括敏感信息误收录、未成年人隐私、恶意链接、供应链风险和页面脚本问题。

## 请报告的问题

- 非公开个人信息、联系方式、家庭住址、证件信息或私人社媒内容被收录。
- 青年球员资料存在明显隐私、骚扰或误导风险。
- 来源链接指向恶意软件、钓鱼页面或已被劫持的网站。
- 页面脚本、依赖或 GitHub Actions 存在可利用风险。
- 公开资料被错误解释为官方、全量或实时数据库，可能造成重大误导。

普通数据错误、死链和口径争议可以直接开 issue；涉及隐私或安全风险时，请不要在公开 issue 中贴敏感细节。

## 报告方式

优先使用 GitHub 的 [Private vulnerability report](https://github.com/starryjog/football-research-sandbox/security/advisories/new)。如果该入口当前不可用，请先通过维护者主页确认私密联系方式；不要改用公开 issue 粘贴敏感正文、密钥或未成年人的非公开资料。

只提供定位问题所需的最小信息：

- 问题类型。
- 涉及文件、页面或链接。
- 为什么有隐私或安全风险。
- 建议的最小修复方式。

普通球员资料纠错、公开来源失效和页面显示问题分别使用仓库 issue 模板；一旦内容涉及私人联系方式、家庭住址、证件、账号凭证或可利用漏洞，就转为私密报告。

## 处理原则

- 优先移除或降级敏感、不可公开验证或可能造成伤害的内容。
- 保留必要的审计说明，但不重复传播敏感材料。
- 第三方来源存在风险时，先删除或替换链接，再补核验备注。
- 如果提交历史中出现密钥或非公开个人信息，除删除当前文件外，还要评估吊销凭证、清理历史和限制继续传播的必要性。
- 数据口径变更同步更新 `docs/scope.md`、`docs/source-policy.md` 或 `CHANGELOG.md`。
